Web Servers

The term index is referring to the default page that prints the structure of hierarchy in a server. For example, opening a textbook to a glossary (otherwise known as index) will give a general overview of where all the different topics are located within the book. This serves as a starting point when searching for a resource with no direct path specified. The index can be named arbitrarily and serves as a general homepage for the website in question.

When the HTML file is viewed from the local computer, there is one main file listed (with lost JavaScript and CSS files). In an alternate fashion, the web server tab contains an executor script as well as a favicon script. This is due to the file being on located on Oregon State file servers. In addition, there is a stark difference between the request and response headers. This is because in the web server variant, there is communication between the browser, web server, and the file server. The copy on the hard disk is located on the computer itself, so the communication is only between the browser and the local computer.

The favicon.ico file has a 200 status code because the get request was successful to retrieve the file from the server. In this case, the response body has the appropriate information. In contrast, no modified CSS or JavaScript files were uploaded to the server and thus, the get request has a fundamental error. The 404 status code specifies that the resource is missing from the server. This is the correct expectation under the given circumstances.

The scheme of a URL displays the protocol the web client is using for the request. In this case, the URL begins with “https.//”, which is a secure version of the hyper text transfer protocol. Next, a subdomain is used as a prefix to a section of a general domain. In this page, the subdomain is “web.engr.” which is linked to an IP address of the Oregon State engineering servers. The host domain is “oregonstate.edu” which is the human friendly name of the engineering servers to which our request must be sent. The last section of a URL must be the resources which provides a path to the requested file. In this case, “~biswasr/a1-biswasr/” leads from the server space designated for the webpage to the folder in which the destination file is located.

Frontend Design

Frontend design is the forefront of usability when it comes to constructing websites and web applications. A well-designed frontend has updated elements such as graphical user interface, navigation links, colors and themes, as well as a thoughtful page layout. Given each annual progression of HTML standards, it’s important to maintain modern interpretations of .css styling on webpages. A poorly designed frontend will frustrate users from accomplishing their goals and ultimately forward their efforts to an alternative website or web application. Both clients and users should expect form and function that makes sense on a website.

Effective
The effectiveness of a website is determined by the success rate of users accomplishing their intended purpose when interacting with the interface.
Efficient
Part of the process of reaching a goal requires efficiency. Users need to be able to fulfill a desire with the least amount of effort or steps. This also means that they can achieve results in a timely manner.
Easy to navigate
For first time users, navigating the website should be uncomplicated. Users should be able to find their resource by relatively easy, and also be able to remember the pathways for repetition.
Error-Free
Websites should be accessible to any of the users who are intended to be the base users. There should not be common errors for users who are trying to submit specific request types.
Enjoyable
Enjoyment in using the website is a key to retaining traffic. The website must be adhered to the target audience in order for users to gain the most utility. The overall experience includes all 4 pillars mentioned above as well as a great frontend design to complete a package.

Page layout tags are important for structuring the content within the body of an .html document. The tags serve a purpose for allowing both the interpreter (browser) and the user to gain an enhanced sense of hierarchical organization. The first essential element is a header. Headers are used to cast a masthead for a website, giving it a more professional introduction to the user. It includes name, publisher, and other arbitrary information. Traditionally, the nav element is presented next with the sole purpose of providing a map for resources. Nav elements can help guide the user from page to page, giving a compass for reaching their intended destination. Once the user selects a resource of interest, they are likely to encounter a main element. The main element holds the significant breadth of content within a webpage. This includes the stories, media, and other primary content. The content is usually broken down by the section element to group together related content. It reduces the time it takes for a user to find the relevant resources in a webpage. Within a section is also subsections defined by the article element. Articles are independent bodies dedicated to a single topic. This can be accessed by navigation elements, further simplifying the path to a user goal. The very last piece to a set of elements is called a footer. The footer is separate from the main element and provides the critical information to a website, including copyright information, contacts, and publication information.

Anchors are a useful tool in HTML construction. Anchors are typically reserved for creating hyperlinks between two resources. There are really two main types of resources: external and internal. Linking to external resources such as other websites uses an absolute path. An absolute path uses a complete path to the resource because it is outside the website. An example of this would be a URL that can be entered in the address bar on a browser. The opposite is true of a resource that shares the same directory as the web or application. Because the link to the resource is in the same root files, a relative path is used to identify it. In general, the relative paths are shorter due to the shared directory. There are three main categories for utilizing anchors:

  1. Internal anchor: This type of anchor provides useful navigation measures for locating specific articles or data within the webpage. These anchors address ID tags that are used in accordance with articles to identify where to take the user to.
  2. External anchor: The best way to support content on a website requires referencing another resource. In this case, an external anchor can be used to hyperlink a resource outside the webpage domain. URL’s can be used to locate the desired resource, with a simplified description as the button for entrance. Because the resource is now located outside of the local directory, an absolute path (URL) must be used.
  3. Page-to-page: One of the most commonly used navigational gestures include accessing different pages within the same website. This gesture is better known as page-to-page referencing. As the name suggests, there is a link which provides a path to each individual page (separate HTML files). These links are based on the HTML file names in the same directory and thus use a relative path to establish a connection.

Optimizing images

Images that are placed in an HTML document must have 6 crucial elements: a descriptive file name, a small file size, exact dimensions, correct file format, reduced resolution, and the proper color mode. A descriptive file name improves search engine optimization to help it sort the image into search results for a similar target. It’s important to be concise while being still descriptive to be a proper name. As mentioned in the frontend design article, there is a time constraint on user attention when it comes to loading content. Thus, it’s critical to reduce the file size to an appropriate resolution to find a balance between loading times and image quality. Typically, lossless compression is the best choice because it does not significantly deteriorate the image quality. Just like with compression, it’s also essential that an image be the proper dimensions to fit the webpage. An overly large file dimension will most definitely increase the loading time because it is not tailored to the viewport. Computers have many formats for images because they all have different properties that provide unique utility from each type. Most online formats take the form of .JPG however, there are line-art images that rely on .GIF or .PNG for transparency. As per each monitors' capability, there are several densities of pixel arrays that can support higher resolution images. The industry standard was 72 pixels per inch, however, there are far more advances displays. This just implies that the use of various image sizes can be useful for catering to different display resolutions. The very last specification is to use the correct color mode per media type. RGB is most commonly used with .PNG, .JPG, .SVG and .WebP Indexed is more often reserved for .GIF or .PNG files.

For photos, a JPG file is the most appropriate choice due to its color properties. For an image to be pleasing, the image must be sharp and colorful. Typically, JPG files can display millions of colors and have fantastic compression properties which allow them to retain quality while reducing load times and saving server space. JPG files don’t have an alpha channel and thus aren’t useful for line art. For this reason, PNG formats are more commonly seen with line art. This is because they have RBG color mode, support compression, and can be placed over just about anything without distortion. The main supporting quality for the PNG format is the alpha channel, which can allow the background to become transparent and thus become an overlay anywhere in the webpage.

Favicons

Favicons are used to display a company logo in a webpage, tab, or bookmarks list. These icons can be in .GIF, .PNG, .ICO, and .SVG file formats. Regardless of the file type, the icon is simplified through a favicon generator and saved in a variety of sizes. All the sizes are usually linked in the head of a website, where the browser can pick up the appropriate size and display it wherever the website is saved or displayed. This icon can also be used in a header to display the logo next to the title of the website. Nowadays, the favicon in the header is commonly used a button for returning to the index of the website.

Cascading Style Sheets

A default HMTL document has a default style, which is monotone and uninspiring, intentionally. Cascading Style Sheets (.css files) are important to incorporate in an HTML file for many reasons other than aesthetics. A more colorful and stylized website can improve usability, legibility, and a dynamic method of restructuring the elements in the document itself. Apart from these ideas, there’s also an importance of load times and maintenance that CSS files can help with. A real-world example is the change in brand requirements from the early days of skeuomorphic design to a more flat, material design invoked standard. As a developer, it’s critical to deliver an end product that looks and feels modern while catering to the brand requirements. Cascading Style Sheets has become the industry solution to creating a fully functional and aesthetically pleasing website.

There are 5 existing methods to incorporate style to an HTML document. The most efficient method is to link an external style sheet in the global head area of an HTML document. Because all HTML documents can link to a global CSS file, this method can provide an efficient way to spread the selected style to all pages of a website. As per updating specific modules or components of a website, other CSS files can be imported into the global stylesheet by using the “@import” function or by linking in the header after the global stylesheet. Embedding styles does not require linking however, it’s mostly for singular purposes because it is rather tedious to repeat this function to entire HTML documents. Embedding is done using the “<style>” tags and can define the appearance of elements such as headings or article text. Inline styling is also similar to embedding but is done directly inside the element tags. For example, you can define an article with blue text using: “<article style = “color blue”><article color= “blue”/>”. This method is rather outdated and still tedious to implement. The very last incorporating method is by exploiting the Document Object Model (DOM) by specifying which element to target and which style to incorporate.
